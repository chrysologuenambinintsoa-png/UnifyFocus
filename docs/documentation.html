Documentation — UnifyFocus

Guide rapide, API, FAQ et contact

Vue d'ensemble

UnifyFocus est une interface web qui fournit des outils de génération de contenu assistés par l'IA, un éditeur, un chat IA, gestion d'utilisateurs et un système de crédits/abonnements.

Getting started

  1. Clonez le dépôt et installez les dépendances.
  2. Copiez le fichier d'environnement `.env.example` -> `.env.local` et renseignez les clés (Stripe, DB, etc.).
  3. Lancer en développement :
npm install
npm run dev

Structure importante

  • src/app — routes/app pages (Next.js app router)
  • src/components — UI, vues et layouts
  • src/lib — utilitaires (ai, db, notify, etc.)
  • src/app/api — endpoints serveur (app routes)
  • prisma/ — schéma et migrations
  • docs/ — pages de documentation et ressources

Commandes utiles

npm run build
npm run start
npm run db:migrate

Principales vues et actions

  • Éditeur IA — création et personnalisation de prompts.
  • Chat — conversation interactive avec l'assistant.
  • Profil — gestion de compte et abonnement (Stripe).
  • Usage — historique de crédits et consommation.

API (exemples)

Endpoints exposés via src/app/api. Exemples :

GET  /api/user/generations?userId=...     # Lister generations (query userId)
DELETE /api/user/generations                 # Supprimer generation(s) (body: { id?, userId })
POST   /api/support                          # Envoyer message support (body: { userId, message })
POST   /api/user/subscription/create-checkout-session  # Créer session Stripe
GET    /api/notifications?userId=...         # Lister notifications
POST   /api/auth/login                        # Auth (selon implémentation)

Détails et exemples

Authentification : la plupart des endpoints utilisateurs supposent un cookie de session ou un token côté serveur. Pour tester en local, utilisez l'auth intégrée ou simulez userId dans les requêtes.

Récupérer les générations (exemple)

curl -s "http://localhost:3000/api/user/generations?userId=USER_ID"
HTTP/1.1 200 OK
{
  "generations": [
    { "id": "g1", "type": "text", "prompt": "...", "status": "completed", "createdAt": "..." }
  ]
}

Supprimer une génération (exemple)

curl -X DELETE "http://localhost:3000/api/user/generations" \
  -H "Content-Type: application/json" \
  -d '{ "id": "g1", "userId": "USER_ID" }'
Response: { "deleted": 1 }

Envoyer un message au support

fetch('/api/support', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ userId: 'USER_ID', message: 'Mon problème...' })
})

Réponse: { "ok": true, "message": "Received" }

Créer une session Stripe (exemple)

fetch('/api/user/subscription/create-checkout-session', { method: 'POST' })
Response: { "url": "https://checkout.stripe.com/..", "id": "cs_test_..." }

Codes d'erreur courants

  • 400 — Requête invalide (champs manquants).
  • 401 — Non authentifié / accès refusé.
  • 403 — Pas d'autorisation pour la ressource.
  • 500 — Erreur serveur.

Webhooks (exemples)

Si vous activez Stripe webhook, configurez l'endpoint correspondant côté serveur pour recevoir les événements de paiement et mettre à jour le paiement/plan de l'utilisateur.

POST /api/webhooks/stripe  # Recevoir events Stripe
Example event body: { "type": "checkout.session.completed", "data": { ... } }

Dépannage

  • Page blanche après clic : ouvrir la console du navigateur (F12) et regarder les erreurs réseau/JS.
  • Import manquant (ex. @/lib/notify) : vérifier l'existence du fichier et exécuter npm run build.
  • Problèmes Stripe : validez les variables d'environnement STRIPE_SECRET_KEY et NEXT_PUBLIC_APP_URL.
  • Logs serveur : consultez la sortie du process Node (terminal où tourne npm run dev).
